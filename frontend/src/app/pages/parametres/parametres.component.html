❓ Comment obtenir ces clés Stripe ?

Stripe permet d'encaisser les paiements par carte en ligne. Créez un compte gratuit sur stripe.com.

1. Clé secrète

  1. Connectez-vous au Dashboard Stripe → Développeurs → Clés API.
  2. En haut à droite, choisissez le mode Test (clés sk_test_…) pour essayer, ou Production (clés sk_live_…) pour encaisser réellement.
  3. Dans « Clé secrète », cliquez sur Révéler puis copiez la valeur (commence par sk_).
  4. Collez-la ci-dessus et enregistrez.

2. Secret du webhook (pour confirmer les paiements)

  1. Allez dans Développeurs → Webhooks → Ajouter un endpoint.
  2. URL de l'endpoint à renseigner : {{ webhookUrl }} (doit être joignable publiquement depuis Internet).
  3. Événement à écouter : checkout.session.completed.
  4. Après création, ouvrez l'endpoint → Secret de signature → copiez la valeur (commence par whsec_) et collez-la ci-dessus.

💡 Astuce : commencez en mode Test avec la carte 4242 4242 4242 4242 (date future, CVC quelconque) pour vérifier que tout fonctionne sans argent réel.

❓ Comment configurer l'envoi avec Gmail ?

Gmail est gratuit et fonctionne bien pour quelques dizaines d'emails par jour, sans nom de domaine.

1. Créer un « mot de passe d'application »

⚠️ Ce n'est pas votre mot de passe Gmail habituel.

  1. Compte Google → Sécurité → activez la Validation en 2 étapes.
  2. Puis Sécurité → Mots de passe des applications.
  3. Donnez un nom (ex. « Restauration ») → Google génère 16 caractères.

2. Remplir le formulaire ci-dessus

  • Serveur SMTP : smtp.gmail.com — Port : 587 — TLS : activé
  • Identifiant : votre adresse Gmail complète
  • Mot de passe : les 16 caractères sans espaces
  • Expéditeur : Mon Restaurant <mon-resto@gmail.com>
  • Cochez « Activer l'envoi » puis Enregistrer, et faites un test.

💡 Limite Gmail : ~500 emails/jour. Pour plus de volume ou une adresse @votre-domaine, utilisez Brevo ou Mailjet (gratuits aussi) : il suffit de changer le serveur SMTP, l'identifiant et le mot de passe.

❓ Comment obtenir la clé API Mistral ?
  1. Créez un compte sur console.mistral.ai.
  2. Activez un plan / moyen de paiement (l'API est facturée à l'usage, au nombre de jetons).
  3. Allez dans API Keys → Create new key.
  4. Copiez la clé et collez-la ci-dessus (elle ne sera plus réaffichée ensuite).

Ce que Mistral fait ici :

  • Robot fournisseur — lit les pages du portail d'un fournisseur pour y trouver la connexion, le catalogue, les produits et leurs prix, puis rattache chaque produit à un ingrédient. Les sélecteurs trouvés sont mis en cache : une synchro de routine ne coûte aucun appel.
  • Recettes — compose une recette originale à partir de votre référentiel d'ingrédients.
  • Événements — propose le calendrier des événements qui font varier la fréquentation d'une ville.

💡 La clé est chiffrée en base et masquée après enregistrement. ⚠️ Mistral répond à partir de ses connaissances (pas de recherche web) et peut se tromper : chaque fonction vous fait valider avant d'enregistrer quoi que ce soit.

❓ Comment obtenir la clé API Météo-France ?

Récupère les températures heure par heure pour une ville sur un mois ou une année, via le portail data de Météo-France.

  1. Créez un compte sur portail-api.meteofrance.fr.
  2. Souscrivez à l'API « Données climatologiques » (données horaires) — l'offre de base est gratuite.
  3. Dans votre application, récupérez la clé API / token et collez-la ci-dessus.
  4. Renseignez la ville et la période (mois facultatif = toute l'année).

💡 La donnée horaire historique de Météo-France s'appuie sur des stations : la ville sera rapprochée de la station la plus proche. La clé est chiffrée et masquée après enregistrement.
